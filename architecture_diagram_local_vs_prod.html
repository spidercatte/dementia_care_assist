Local vs Production Decoupled Architectures

Comparing the 5-container Docker Compose development setup with the decoupled Google Cloud production stack, highlighting the separate Backend API and MCP Server services.

Local Development (Docker Compose)

Five containers in a shared Docker Compose network. Backend (port 8000) and MCP Server (port 8002) are separate services, both sharing PostgreSQL and ChromaDB via the common library.

Production Environment (Google Cloud)

Decoupled cloud stack. Backend API and MCP Server are separate Cloud Run services sharing Cloud SQL and Vertex AI Search. No ChromaDB in production — Vertex AI Search handles all semantic search.
